What are the best practices associated with using classes vs. ids?


In using a common CSS makes layout and formatting of your website pages, especially if it has more than one page or has a complex layout or formatting.


This is because if you have uniform styles for the content of the website but things like text size,emphasis like bold, italic and other styling options you may want to use need to be applied in the web page at the point they occur.


This is not a problem per se if the website doe's not change or change often, but presents extra effort where the change applies to multiple parts of the page content. So for example you use a particular style for paragraphs for technical details and there are several places and many pages where technical details are present. The way you would normally apply the change is to find every paragraph and every page technical content is present that the current style was applied to and update the style for each paragraph. This task would involve alot effort applying the same change, just to different parts of the website.


Thanks to CSS, there is a more efficient way to manage the application and updating of styles and layout in your website.


We can use class'es and id's to act as identifiers for elements that are to have the same type of formatting or positioning, across your website. These identifiers are added to the content tags where appropriate.


If we use the example from above where technical content all have the same style, then it might look something like this,

if it were a class <p class="technical_content"> Some technical text</p> or if were an id <p id="technical_content"> Some technical text<p>

, in our CSS We would have an entry that defined the "technical_content" class, something like this..


The '.' is used to prefix the class in the CSS .technical_content{ font-family: Arial; font-style: italic;}

the '#' is used to prefix id's in the CSS #technical_content{ font-family: Arial; font-style: italic;}

We add the .technical_content class to any/every paragraph that contains technical content, this will format each of those the same way now as it was before. The big difference now is that if we want to make changes to the technical content style in future we only have to make the change in the technical_content class definition in our CSS and magically the new styling is applied everywhere this class was applied.


Classes and id's are both used to identify web page elements the key difference and reason for choosign the right one to use or define is whether or not the attributes you are wanting to apply is used only once on the web page or more than once. A particular class can be used more than once on a web page. An id can only be used once on each web page.Each page will only have one footer, one header, one search element so these would be best defined using id's, many elements of your page though may be the same font family, might be italic, might be centered so these elements would be best defined using class'es


id's are unique in a page, there is only one of you, many people can belong to a class, many people have black hair, many people wear glasses.
